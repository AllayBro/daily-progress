#  Daily Progress - Ежедневный журнал прогресса

Эта программа помогает вести ежедневный журнал того, что ты сделал и что не успел сделать. Она автоматически создает файлы, коммитит изменения в Git и отправляет их в репозиторий.

> ** Быстрый доступ:** [COMMANDS.md](COMMANDS.md) - все команды в одном месте!

##  ПОШАГОВАЯ ИНСТРУКЦИЯ ПО ИСПОЛЬЗОВАНИЮ

### Шаг 1: Подготовка (только один раз)

#### 1.1 Открой PowerShell

**Способ 1 (быстрый):**

- Нажми `Win + R` (клавиша Windows + R)
- В появившемся окне введи: `powershell`
- Нажми Enter

**Способ 2 (через меню):**

- Нажми клавишу Windows
- Введи "PowerShell"
- Нажми Enter на "Windows PowerShell"

#### 1.2 Перейди в папку с программой

Скопируй и вставь эту команду в PowerShell:

```powershell
cd "C:\Users\AllayBro\Desktop\УЧЕБА\ПО ГИТ и ГИТХАБ\Практика\Что я выполнил сегодня"
```

**Что произойдет:** Ты увидишь, что путь в начале строки изменился на:

```
PS C:\Users\AllayBro\Desktop\УЧЕБА\ПО ГИТ и ГИТХАБ\Практика\Что я выполнил сегодня>
```

#### 1.3 Проверь, что Git настроен

```powershell
git config user.name
git config user.email
```

**Если покажет твое имя и email** - все готово!

**Если покажет ошибку** - нужно настроить Git (см. раздел "Проблемы").

### Шаг 2: Ежедневное использование

#### 2.1 Запусти программу

```powershell
.\daily-commit.ps1
```

#### 2.2 Что произойдет

1. **Программа создаст файл** для сегодняшней даты (если его еще нет)
2. **Откроется редактор** с этим файлом
3. **Ты заполнишь файл** (см. примеры ниже)
4. **Вернешься в PowerShell** и введешь название коммита
5. **Программа автоматически** закоммитит и отправит изменения

#### 2.3 Заполни файл

В открывшемся файле ты увидишь такую структуру:

```markdown
# 2025-08-30

## Done

- Что ты сделал сегодня
- Какие задачи выполнил
- Какие цели достиг

## Not done

- Что не успел сделать
- Что отложил на завтра
- Какие проблемы возникли
```

**Пример заполнения:**

```markdown
# 2025-08-30

## Done

- Изучил основы Git и GitHub
- Создал первый репозиторий
- Написал простой скрипт для автоматизации
- Прочитал главу по PowerShell

## Not done

- Не успел изучить ветки в Git
- Отложил изучение merge конфликтов
- Нужно больше практики с командной строкой
```

#### 2.4 Сохрани файл и вернись в PowerShell

1. **Сохрани файл:** Нажми `Ctrl + S`
2. **Закрой редактор:** Нажми `Alt + F4` или крестик
3. **Вернись в PowerShell:** Ты увидишь сообщение "Введите название коммита:"

#### 2.5 Введи название коммита

Напиши краткое описание того, что ты сделал сегодня. Например:

```
Progress: 2025-08-30  Изучил Git и создал первый репозиторий
```

**Примеры хороших названий:**

```
Progress: 2025-08-30  Изучил основы Git и GitHub
Progress: 2025-08-30  Создал систему ведения дневника
Progress: 2025-08-30  Исправил ошибки в скрипте
```

#### 2.6 Готово

Программа автоматически:

- Создаст коммит с твоим сообщением
- Отправит изменения в GitHub
- Покажет результат операции

##  РЕШЕНИЕ ПРОБЛЕМ

### Проблема 1: "No items under 'Done'. Add at least one '- ...'."

**Причина:**

- Ты не заполнил секцию "Done"

**Решение:**

1. Открой файл для сегодняшней даты
2. Убедись, что в секции "Done" есть хотя бы один пункт:

```markdown
## Done

- Что-то сделал сегодня (обязательно!)
```

**Пример правильного заполнения:**

```markdown
## Done

- Прочитал документацию
- Установил программу
- Создал первый файл
```

### Проблема 2: "Git не настроен!"

**Причина:**

- Git не был настроен после установки

**Решение:**

1. Выполни эти команды в PowerShell:

```powershell
git config --global user.name "Твое Имя"
git config --global user.email "твой@email.com"
```

**Пример:**

```powershell
git config --global user.name "AllayBro"
git config --global user.email "artemzotov007@inbox.ru"
```

4. Проверь, что настройки сохранились:

```powershell
git config user.name
git config user.email
```

### Проблема 3: "Push error!" или "Exception on push!"

**Причины:**

- Нет интернета
- Проблемы с GitHub
- Конфликт изменений

**Решение:**

1. **Проверь интернет:** Открой браузер и зайди на любой сайт
2. **Попробуй вручную:**

```powershell
git pull
git push
```

3. **Если не помогает, попробуй:**

```powershell
git pull --rebase
git push
```

### Проблема 4: "Nothing to commit."

**Причина:**

- Ты не изменил файл

**Решение:**

1. Открой файл для сегодняшней даты
2. Добавь хотя бы один пункт в секцию "Done"
3. Сохрани файл
4. Запусти программу снова

### Проблема 5: Файл не открывается в редакторе

**Решение:**

1. Найди файл вручную в папке `journal\2025\08\`
2. Открой его в любом текстовом редакторе
3. Заполни и сохрани
4. Вернись в PowerShell

### Проблема 6: "Permission denied" или "Access denied"

**Решение:**

1. Закрой PowerShell
2. Нажми правой кнопкой на PowerShell
3. Выбери "Запуск от имени администратора"
4. Повтори все шаги

##  СОВЕТЫ ПО ВЕДЕНИЮ ЖУРНАЛА

### Как лучше вести журнал

1. **Пиши конкретно:** Вместо "Изучал Git" напиши "Изучил команды git branch и git merge"
2. **Отмечай прогресс:** Записывай не только что сделал, но и чему научился
3. **Будь честным:** Если ничего не делал, так и пиши
4. **Планируй:** В секции "Not done" записывай планы на завтра

### Примеры хороших записей

```markdown
## Done

- Изучил команды git branch, git checkout, git merge
- Создал 3 ветки и успешно объединил их
- Решил конфликт при слиянии файлов
- Написал документацию к проекту (5 страниц)
- Прочитал главу "Git Workflow" в книге

## Not done

- Не успел изучить git rebase (планирую завтра)
- Отложил изучение GitHub Actions до следующей недели
- Нужно больше практики с удаленными репозиториями
- Хочу создать свой первый open-source проект
```

### Примеры плохих записей

```markdown
## Done

- Работал
- Учился
- Делал что-то

## Not done

- Много всего
- Не знаю
- Что-то там
```

##  ЧАСТО ЗАДАВАЕМЫЕ ВОПРОСЫ

### Вопрос: Могу ли я использовать программу на другом компьютере?

**Ответ:** Да, но нужно будет заново настроить Git и скопировать файлы программы.

### Вопрос: Что делать, если я забыл записать что-то вчера?

**Ответ:** Просто открой файл за вчерашнюю дату и добавь записи. Программа закоммитит изменения.

### Вопрос: Могу ли я изменить формат файлов?

**Ответ:** Да, но нужно будет изменить скрипт программы.

### Вопрос: Что делать, если программа не работает?

**Ответ:**

1. Проверь все требования из раздела "Проверки"
2. Убедись, что ты в правильной папке
3. Проверь, что Git настроен
4. Попробуй перезапустить PowerShell

### Вопрос: Могу ли я посмотреть свои записи за прошлые дни?

**Ответ:** Да! Все записи сохраняются в папке `journal\2025\08\` в виде файлов `.md`.
